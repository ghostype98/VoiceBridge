# 端口配置说明（唯一修改处）

**以后更换端口只需改一个文件：`config/config.yaml`**

在本文件中修改以下三处，并保持 **主服务两处一致** 即可：

| 配置路径 | 说明 | 默认值 |
|----------|------|--------|
| `voice_streaming.websocket.port` | WebSocket 语音流式服务端口 | 8003 |
| `services.voicebridge.port` | 主服务端口（API + 前端一体） | 8002 |
| `server.port` | 与 `services.voicebridge.port` 保持一致 | 8002 |

修改后：

1. **手动运行**：重启主服务 `bash bash/restart_service.sh`；NATAPP 执行 `bash tools/natapp/start.sh`（脚本会从本配置读取主端口并写入 `tools/natapp/config.ini`）。
2. **systemd 开机自启**：重启主服务 `sudo systemctl restart voicebridge`，重启内网穿透 `sudo systemctl restart natapp`。

当前项目内所有端口引用（启动脚本、日志输出、NATAPP）均以本配置文件为准。
