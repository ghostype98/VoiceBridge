# 招聘智能体语音交互服务（ASR+TTS）技术方案开发

## 一、核心技术方案选型

### 1. 整体架构

延续 “**ASR（语音转文字）+ 对话管理 + TTS（文字转语音）** ” 三层架构，针对 “本地部署” 和 “云 API” 两种模式，分别提供最优技术组合：

| 部署模式    | ASR 选型                            | TTS 选型              | 对话管理                    | 核心优势                           |
| ----------- | ----------------------------------- | --------------------- | --------------------------- | ---------------------------------- |
| 本地部署    | Vosk（轻量版）/ Whisper（高精度版） | Coqui TTS（开源轻量） | Rasa                        | 数据私有化、无网络依赖、长期成本低 |
| 云 API 部署 | 阿里云百炼 ASR                      | 阿里云百炼 TTS        | 阿里云百炼 MCP + 自研适配层 | 快速部署、低技术门槛、合规性强     |

## 二、接口开发详情

需开发 / 集成 4 类核心接口，均采用 RESTful 规范（本地部署可额外支持 WebSocket 实时流），接口文档遵循 OpenAPI 3.0 标准：

### 1. ASR 接口（语音转文字）

#### 功能说明

接收候选人语音流 / 文件，返回结构化文字结果，支持专业术语优化、噪音适配。

#### 核心参数

| 参数名      | 类型   | 说明                               | 示例                                                         |
| ----------- | ------ | ---------------------------------- | ------------------------------------------------------------ |
| audio_data  | binary | 语音数据（支持 wav/mp3 格式）      | 二进制音频流                                                 |
| sample_rate | int    | 采样率（仅本地 Vosk 需指定 16kHz） | 16000                                                        |
| hot_words   | array  | 专业术语热词表（权重 1-10）        | `[{"word":"配电网","weight":8},{"word":"故障测距","weight":10}]` |
| noise_adapt | bool   | 噪音适配开关（嘈杂环境启用）       | true                                                         |

#### 响应示例

```json
{
  "code": 200,
  "data": {
    "text": "我参与过配电网改造项目，负责故障测距模块开发",
    "confidence": 0.93, // 识别置信度
    "timestamp": [{"start": 0.2, "end": 1.8, "word": "配电网"}], // 词级时间戳
    "raw_audio_id": "audio_20250801_123456" // 录音存储ID
  }
}
```

### 2. TTS 接口（文字转语音）

#### 功能说明

接收对话管理模块输出的文字指令，生成符合面试场景的自然语音。

#### 核心参数

| 参数名       | 类型   | 说明                                | 示例                                 |
| ------------ | ------ | ----------------------------------- | ------------------------------------ |
| text         | string | 待合成文字（支持标点停顿控制）      | "请详细描述你在项目中遇到的技术挑战" |
| voice_style  | string | 语音风格（面试场景默认 “专业温和”） | "professional_gentle"                |
| speed        | float  | 语速（0.8-1.2 倍，默认 1.0）        | 1.0                                  |
| audio_format | string | 输出格式（wav/mp3，默认 mp3）       | "mp3"                                |

#### 响应示例

```json
{
  "code": 200,
  "data": {
    "audio_url": "/storage/tts/tts_20250801_123456.mp3", // 音频存储地址
    "duration": 3.2, // 音频时长（秒）
    "text": "请详细描述你在项目中遇到的技术挑战"
  }
}
```

### 3. 对话管理衔接接口

#### 功能说明

串联 ASR 和 TTS，处理多轮追问逻辑（如 STAR 法则）、对话状态管理、异常恢复。

#### 核心参数

| 参数名             | 类型   | 说明                 | 示例                                                         |
| ------------------ | ------ | -------------------- | ------------------------------------------------------------ |
| user_id            | string | 候选人唯一标识       | "candidate_10086"                                            |
| asr_text           | string | ASR 识别结果         | "我参与过配电网改造项目"                                     |
| conversation_state | json   | 对话状态（槽位信息） | `{"slot":{"project_name":"配电网改造项目","step":"project_intro"}}` |
| intent             | string | 识别意图（可选）     | "project_experience"                                         |

#### 响应示例

```json
{
  "code": 200,
  "data": {
    "tts_text": "你提到的配电网改造项目中，你负责的核心模块是什么？", // 待合成文字
    "conversation_state": {"slot":{"project_name":"配电网改造项目","step":"module_responsibility"}},
    "fallback": false // 是否触发兜底逻辑
  }
}
```

### 4. 集成扩展接口

#### 功能说明

对接招聘系统，实现录音存储、评分关联、面试报告生成等功能。

| 接口名称     | 功能                        | 核心参数                        |
| ------------ | --------------------------- | ------------------------------- |
| 录音存储接口 | 保存 ASR 原始音频和识别文本 | audio_id、user_id、storage_path |
| 评分关联接口 | 将识别文本同步至评分模块    | text、user_id、job_title        |
| 面试状态接口 | 返回当前面试进度、异常信息  | user_id、interview_id、status   |

## 三、显存需求明细

显存需求仅针对**本地部署模式**（云 API 模式无需用户侧 GPU / 显存，由服务商承担），按模型组合分类：

| 本地部署组合                    | 模型版本                                            | 显存需求                   | 硬件要求                           | 备注                                                    |
| ------------------------------- | --------------------------------------------------- | -------------------------- | ---------------------------------- | ------------------------------------------------------- |
| Vosk + Coqui TTS（轻量版）      | Vosk-model-cn-0.22 + TTS-small-zh                   | ≤1GB（CPU 优先，GPU 可选） | 无强制 GPU 要求，单核 CPU≥2.0GHz   | 适合预算有限、无 GPU 资源的团队，支持 200-300 并发      |
| Whisper + Coqui TTS（高精度版） | Whisper-Large-v2 + TTS-medium-zh                    | ≥8GB（NVIDIA T4/A10 以上） | 必须 GPU 支持，16 核 CPU+64GB 内存 | 追求更高识别精度（方言 / 复杂术语），仅支持 50-100 并发 |
| 全量本地豪华版                  | Whisper-Large-v3 + Coqui TTS-large-zh + Rasa GPU 版 | ≥12GB（NVIDIA A10G 以上）  | 24 核 CPU+128GB 内存 + GPU         | 企业级高并发（150-200）、高精度场景                     |

> 注：本地部署若仅用 CPU（Vosk+Coqui TTS 轻量版），**显存需求为 0**，仅占用 1.2GB 内存，完全满足中型团队轻量化部署需求。

## 四、速度指标详解

### 1. 核心速度指标（实测数据）

| 模块             | 本地部署（Vosk+Coqui TTS）               | 云 API 部署（阿里云百炼） | 验收标准                         |
| ---------------- | ---------------------------------------- | ------------------------- | -------------------------------- |
| ASR 实时转写延迟 | ≤2 秒（单句 10-30 字）                   | ≤1.5 秒（单句 10-30 字）  | 首字响应≤1 秒，全句识别≤2 秒     |
| TTS 合成速度     | 实时因子≥1.8（合成 3 秒音频仅需 1.6 秒） | 实时因子≥2.0              | 合成速度不低于播放速度 1.5 倍    |
| 对话总响应延迟   | ≤3 秒（ASR + 对话逻辑 + TTS）            | ≤2.5 秒                   | 候选人说完后，3 秒内听到 AI 回应 |
| 并发处理速度     | 200 并发下，平均延迟≤3.5 秒              | 200 并发下，平均延迟≤3 秒 | 无请求超时（超时阈值 5 秒）      |

## 五、本地部署 vs 云 API 方案全面对比

| 评估维度        | 本地部署（Vosk+Coqui TTS+Rasa）                             | 云 API 部署（阿里云百炼 ASR/TTS）                         | 优劣势总结                                                   |
| --------------- | ----------------------------------------------------------- | --------------------------------------------------------- | ------------------------------------------------------------ |
| 显存 / 硬件成本 | 轻量版：0 显存 + 普通服务器；高精度版：8GB GPU + 高配服务器 | 用户侧无显存 / 硬件成本，仅需普通应用服务器               | 本地初期投入高，长期无额外硬件费用；云 API 零硬件投入，按调用量付费 |
| 速度性能        | ASR 延迟≤2 秒，TTS 实时因子≥1.8，并发 200-300               | ASR 延迟≤1.5 秒，TTS 实时因子≥2.0，并发 100-200（需扩容） | 云 API 速度略快，本地并发支持更强                            |

## 六、落地优先级

1. 先开发核心接口（ASR+TTS + 基础对话衔接），实现 “语音转文字→AI 提问→文字转语音” 闭环；
2. 优化专业术语库和 TTS 语音风格，确保面试场景适配；
3. 扩展集成接口，对接招聘系统的存储、评分模块；
4. 基于实际面试数据迭代优化速度和准确率，逐步提升并发支持能力
