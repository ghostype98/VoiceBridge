## 1. 项目全局技术基线

### 1.1 技术栈

- 后端框架：`FastAPI`（主入口 `app/main.py`）
- Web 服务：`uvicorn`
- 数据库访问：`psycopg2` + `ThreadedConnectionPool`（`app/database/connection.py`）
- 数据库：`PostgreSQL`（默认配置见 `app/database/config.py`）
- 前端：`HTML + CSS + 原生 JavaScript`（`frontend/`）
- 实时语音链路：浏览器 `WebSocket` -> 后端 `ws/asr` -> 阿里云 ASR（`app/voice_streaming/`）
- 评分与评估：本地/兼容 OpenAI 协议 LLM（`app/services/llm_service.py`）
- 面试单题评分 Agent：`agent/evaluation_service.py`
- 面试 21 维评估 Agent：`agent/interview_evaluation_service.py`

### 1.2 服务形态

- 单端口主服务：后端 API + 前端静态页面同端口提供（`app/main.py` 中 `StaticFiles` 挂载）
- WebSocket ASR 入口：
  - FastAPI 暴露：`/ws/asr`（在主端口上）
  - 内部流式服务端口配置：`voice_streaming.websocket.port`（默认 8003）
- 可选脚本存在历史端口（如 8010/8765），但当前基线以 `config/config.yaml` + `services/run.py` + `bash/start_service.sh` 为主。

### 1.3 模块边界

- `app/routers/`：业务 HTTP API 路由
- `app/services/`：应用服务（会话、对话、LLM）
- `app/voice_streaming/`：流式语音服务（ASR、实时评分、WebSocket）
- `app/database/`：数据库连接与数据访问
- `agent/`：评分/评估智能体与提示词
- `frontend/`：登录页、面试页（桌面/移动）
- `bash/`：启动、停止、重启脚本
- `services/`：统一运行入口与服务管理脚本

## 3.4 端到端操作步骤（前端实际行为）

### 步骤 A：登录

1. 用户在 `login.html` 输入用户名密码
2. `login.js` 调用 `window.API.auth.login(username, password)`
3. 登录成功后写入本地：
   - `localStorage.isLoggedIn=true`
   - `localStorage.userData`
   - `localStorage.invitationData`
4. 根据设备跳转：
   - 桌面：`/interview`
   - 移动：`/mobile-interview`

### 步骤 B：面试页初始化

1. `interview.js` 执行 `initializePage()`
2. 本地认证校验 + 后端会话校验 `POST /api/v1/auth/check-session`
3. 拉取 UI 开关：`GET /api/v1/interview/ui-config`
4. 加载用户信息、初始化按钮与状态
5. 自动调用 `startInterview()`

### 步骤 C：开始面试

1. 调用 `POST /api/v1/interview/start`（参数使用 invitation_id）
2. 建立 WebSocket（`frontend/js/websocket-asr.js`）到 `/ws/asr`
3. 发送 `start_recording` 控制消息
4. 页面进入录音与实时转写状态

### 步骤 D：作答与提交

1. WebSocket 持续发送音频块（二进制 PCM）
2. 接收 `intermediate_text` / `final_text` 更新转写 UI
3. 点击“回答完毕下一题”（同开始按钮切换行为）
4. 后端评分并返回：
   - 下一题 / 追问题
   - 得分、追问标记、推理说明

### 步骤 E：结束面试

1. 点击“结束面试”
2. 调用 `POST /api/v1/interview/complete`（`FormData`）
3. 后端触发 21 维评估
4. 返回最终评估结果
5. 前端清理会话资源（含 `beforeunload/pagehide` 防护逻辑）

## 5. Agent 设计基线

## 5.1 单题评分 Agent（`agent/evaluation_service.py`）

- 输入：
  - 问题文本
  - 候选人回答
  - 评估要点（可 JSON 字符串、对象、空）
  - 问题类型 `BASIC` / `SPECIALTY`
  - 可选 `standard_answer`
- 提示词模板：
  - `agent/prompts/evaluation_prompt_basic.md`
  - `agent/prompts/evaluation_prompt_professional.md`
- LLM 调用参数：
  - `temperature=0.1`
  - `max_tokens=800`
- 输出规范：
  - `score` / `grade` / `point_scores` / `reasoning`
  - 容错：解析失败时返回默认及格分 50

## 5.2 追问决策 Agent（`app/services/dialogue_service.py`）

- 核心策略：
  - 仅 `PROFESSIONAL` 题触发 STAR 追问判断
  - 追问上限：单题 1 次
  - 阈值：
    - `>=85`：不追问
    - `<50`：不追问（直接下一题）
    - `50-84`：检测 STAR 缺失后可追问
- 维度优先级：`R > A > T > S`
- 输出动作：
  - `follow_up` 或 `next_question`

## 5.3 面试综合评估 Agent（`agent/interview_evaluation_service.py`）

- 评估维度：21 个维度分组配置 `INTERVIEW_EVALUATION_DIMENSIONS`
- 评估模式：强制分阶段（Map-Reduce，21 维逐维评估）
- 核心流程：
  1. 取会话答案与题目信息
  2. 读取岗位 JD（`job_description_base.core_requirements`）
  3. 维度映射与重要等级更新（必备/加分项/弹性/不重要）
  4. 每维构建 Prompt 单独评估
  5. 合并维度分数，按重要等级加权计算总体分
  6. 生成结构化叙事并落库

## 6.2 当前“RAG”实现形态（代码事实）

当前不是向量库检索式 RAG，而是“数据库条件检索 + Prompt 注入”的准 RAG：

1. 从 `job_description_base` 取岗位 `core_requirements`
2. 解析 21 维名称与重要等级
3. 从 `evaluation_dimension` 表按维度名取说明、关键指标
4. 从 `candidate_answers + interview_question` 取该维度相关题目（按分类过滤）
5. 将检索结果拼装为维度级 Prompt 输入 LLM

## 6.3 准 RAG 的检索来源

- JD 主数据：`job_description_base.core_requirements`
- 维度知识：`evaluation_dimension`
- 面试事实语料：`candidate_answers`、`interview_question`、`interview_questions`

## 7. 全流程步骤（后端视角）

1. 登录校验：查 `interview_invitation`，状态必须 `CONFIRMED`
2. 更新状态：登录成功置 `IN_PROGRESS`
3. 开始面试：创建会话记录，加载题目
4. 建立 WebSocket：开始语音流传输
5. 转写回传：实时文本更新前端
6. 答案提交：写 `candidate_answers`（主）+ 更新 `interview_session`（兼容/进度）
7. 单题评分：调用 `evaluation_service`
8. 追问决策：调用 `dialogue_service`
9. 下一题或追问题：循环
10. 结束面试：会话完成 + 邀请状态 `COMPLETED`
11. 21 维评估：调用 `interview_evaluation_service`
12. 落库 `interview_evaluation_record` 并返回结果

## 9. 模型使用与调用逻辑基线

## 9.1 LLM 服务封装

文件：`app/services/llm_service.py`

- 统一调用路径：
  - 本地/兼容：`{LLM_API_BASE}/v1/chat/completions`
  - 健康检查：`{LLM_API_BASE}/v1/models`
- 关键参数：
  - `model`：来自配置 `settings.LLM_MODEL`
  - `temperature`：配置默认 + 业务覆盖
  - `max_tokens`：配置默认 + 业务覆盖
  - `max_input_tokens`：按上下文长度限制注入

## 9.2 单题评分调用链

`interview_flow.voice-answer` -> `_evaluate_answer_with_llm` -> `evaluation_service.evaluate_answer` -> `llm_service.chat_completion`

## 9.3 实时评分调用链（语音流）

`websocket_server` -> `realtime_scorer.evaluate_answer` -> `llm_service.chat_completion`

## 9.4 21 维评估调用链

`interview_flow.complete` -> `interview_evaluation_service.evaluate_interview` -> 分维度多次 `llm_service.chat_completion` -> 合并后写库

## 9.5 模型使用清单（独立）

### 9.5.1 全局模型配置入口

- 配置文件：`config/config.yaml`
- 配置项：
  - `llm.api_base`
  - `llm.model`
  - `llm.provider`
  - `llm.temperature`
  - `llm.max_tokens`
  - `llm.timeout`
  - `llm.max_retries`
  - `llm.max_context_length`
  - `llm.max_input_tokens`
- 运行时读取：`config/settings.py` -> `Settings`
- 统一调用封装：`app/services/llm_service.py`（`LLMService`）

### 9.5.2 模型用途清单

1. **单题回答评分模型**
   - 用途：对当前题回答进行结构化评分（`score/grade/point_scores/reasoning`）
   - 入口：
     - `app/routers/interview_flow.py` -> `_evaluate_answer_with_llm`
     - `agent/evaluation_service.py` -> `evaluate_answer`
   - 提示词：
     - `agent/prompts/evaluation_prompt_basic.md`
     - `agent/prompts/evaluation_prompt_professional.md`
   - 调用参数（代码基线）：
     - `temperature=0.1`
     - `max_tokens=800`

2. **实时语音评分模型**
   - 用途：语音流式作答过程中实时评分、追问触发判定
   - 入口：
     - `app/voice_streaming/realtime_scorer.py` -> `evaluate_answer`
   - 调用参数（代码基线）：
     - `temperature=0.3`
     - `max_tokens` 取 `voice_streaming.evaluation.llm_scoring.scoring_max_tokens`，默认回落到 `settings.LLM_MAX_TOKENS`
   - 特殊逻辑：
     - 支持截断重试（`retry_on_truncation`）
     - 支持重试 token 放大（`retry_multiplier`）

3. **追问生成模型**
   - 用途：当分数触发条件满足时生成追问问题与追问评估要点
   - 入口：
     - `app/voice_streaming/realtime_scorer.py` -> `_generate_follow_up_question`
     - `app/services/dialogue_service.py`（STAR 维度判断与追问文案生成）
   - 触发策略：
     - 专业题优先
     - 分数与 STAR 缺失联合判定

4. **21维综合评估模型**
   - 用途：面试结束后按 21 维输出 `dimension_scores/dimension_details`
   - 入口：
     - `agent/interview_evaluation_service.py` -> `evaluate_interview`
   - 提示词：
     - `agent/prompts/interview_evaluation_system_prompt.md`
     - 动态拼接 `_build_evaluation_prompt(...)`
   - 调用模式：
     - 分阶段 Map-Reduce（逐维调用）
     - 结果合并后加权计算总体分

5. **评估叙事总结模型**
   - 用途：输出业务可读结论、优势、风险、复试建议
   - 入口：
     - `agent/interview_evaluation_service.py` 的聚合后处理阶段
   - 提示词：
     - `agent/prompts/interview_evaluation_narrative_prompt.md`

6. **核心能力总结模型**
   - 用途：按 JD 重要等级生成“核心能力总结（分节）”
   - 入口：
     - `agent/interview_evaluation_service.py` 聚合链路
   - 提示词：
     - `agent/prompts/interview_evaluation_core_summary_prompt.md`

### 9.5.3 模型输入输出清单

1. **单题评分输入**
   - 问题文本
   - 候选人回答文本
   - 评估要点（JSON/字符串）
   - 问题类型（`BASIC`/`SPECIALTY`）
   - 可选参考答案（专业题）

2. **单题评分输出**
   - `score`
   - `grade`
   - `point_scores`
   - `reasoning`
   - 可选：`need_follow_up`、`key_info`

3. **21维评估输入**
   - 面试问答摘要（按维度过滤后的题目与回答）
   - 岗位要求（`core_requirements`）
   - 维度元数据（`evaluation_dimension`）

4. **21维评估输出**
   - `dimension_scores`
   - `dimension_details`
   - `overall_score`
   - `evaluation_summary`
   - `evaluation_suggestions`
   - 结构化叙事（`evaluation_structured`）

### 9.5.4 模型失败与降级策略清单

1. **单题评分失败降级**
   - 返回默认评分结果（通常 50 分及格）
   - 继续流程，避免中断面试
   - 记录日志供后续追溯

2. **实时评分失败降级**
   - 可返回低风险默认结果并继续下一步
   - 截断时触发一次重试并放大 `max_tokens`

3. **21维评估失败降级**
   - 返回默认综合评估结构
   - 保证 `complete` 接口不因评估失败而整体失败

### 9.5.5 模型健康检查清单

1. 统一健康检查接口
   - `LLMService.health_check()` 请求 `GET {base_url}/v1/models`

2. 运行前检查项
   - `llm.api_base` 可访问
   - `llm.model` 与服务端可用模型一致
   - 超时参数与上下文上限配置合理

3. 故障排查顺序
   - 先看网络连通与接口可达
   - 再看响应是否截断（`finish_reason=length`）
   - 最后看 JSON 解析与提示词格式是否稳定

## 12. 当前代码基线中的注意事项

1. 存在历史脚本与当前配置端口不一致情况（8010/8765 与 8002/8003 并存），部署时必须统一以 `config/config.yaml` + `services/run.py` 为准。  
2. `app/database/config.py` 与 `config/config.yaml` 都包含敏感配置痕迹，生产必须改为密钥托管或环境变量。  
3. 前端 `api.js` 中存在部分历史接口路径（如 `interview-flow`），实际联调用当前路由定义核对。  
4. 评估链路已做多层容错（默认分、重试、截断处理），上线前仍需针对真实数据压测与抽样校验。  
5. 语音、评分、追问、21维评估都依赖外部或本地模型服务稳定性，建议建立健康探针和失败降级策略。
