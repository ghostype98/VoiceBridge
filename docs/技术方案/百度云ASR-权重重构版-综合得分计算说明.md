# 百度云 ASR「权重重构版」综合得分计算说明

本文说明「权重重构版」报告里**综合得分**（0～100）的完整计算过程，与后端 `baidu_asr_portrait_weighted` 相关实现逻辑一致，便于评审、验收与手算核对。

**符号约定**（全文统一，避免依赖公式渲染）：

- `s_i`：第 i 个维度的模型分（0～100，已钳位）
- `M_i`：该维所属「四大模块」在画像中的比例（0～1）
- `L_i`：该维在 JD 上的等级乘子（见第三节）
- `w_i`：合成权重，`w_i = M_i * L_i`

**公式/算式在文件中的形态**：均为反引号符号与 fenced 代码块（纯文本），不是 LaTeX。保存或分发时，**`.md`（Markdown）**、**`.pdf`**、**`.html`**、**`.docx`（Word）** 等常见导出方式一般都能完整保留显示；若需要分式、上下标等印刷级数学排版，需在支持 LaTeX 或公式编辑器的软件中另行编写。

## 一、与普通「百度云 ASR」页的差异

| 项目 | 普通百度云 ASR | 权重重构版 |
|------|----------------|------------|
| 各维分数 `s_i` | 同一次 ASR 的 `dimension_scores` | **相同** |
| 综合分含义 | 语音管线中的简单汇总（如均分类） | **加权平均**：画像四大模块比例 × JD 等级乘子 |
| 画像与 JD | 不参与该综合分 | **参与**（`four_modules` + `importance_level`） |

## 二、计算前的数据准备

### 2.1 各维模型分 `s_i`

- **来源**：同一次百度云 ASR 报告中的 `dimension_scores`（维度名 → 数值）。
- **处理**：维度名经规范化（如去掉前导序号）得到 `norm`；将分数解析为浮点数后**钳位**到闭区间 **[0, 100]**，记为 `s_i`。

### 2.2 JD 重要等级（用于 `L_i`）

- **来源**：由岗位 JD / 维度配置构建的 **`jd_importance_map`**：`norm`（或原始名）→ 该维在岗位上的 **`importance_level` 原文**。

### 2.3 维度所属「四大模块」之一（用于 `M_i`）

由 `evaluation_dimension.category` 映射到画像中的四个键之一：

| category（示例） | 画像模块键 `mod` |
|------------------|------------------|
| 基础素质层面 | 基础素质 |
| 专业能力层面 | 专业能力 |
| 岗位匹配层面 | 岗位匹配 |
| 潜力适配层面 | 潜力文化 |
| 通用评估层面 | 潜力文化 |
| 未配置或未命中 | **潜力文化**（默认） |

对维度 i，记其模块为 `mod(i)`。

### 2.4 四大模块比例一行（`M` 的来源）

从 **`talent_portrait_profile`** 的 `payload.weights.four_modules` 中取**一行**四个比例（每个在 0～1，设计上四者之和为 1）。

**画像查询顺序（与岗位对齐）**

1. 规范化后的 **`(requester, department, position)`**，查一条**启用**的画像；
2. 若无命中：再查 **「仅岗位」**画像——`requester`、`department` 为空，**`position` 为同一岗位名称**；
3. 若仍无：使用系统默认的 **`four_modules.global`**。

**在该画像内选用哪一行比例**

- 若当前面试的主原型（如 `rd_ai` 等）在 `four_modules.archetypes` 中有有效子对象，则用**该原型**对应的一行；
- 否则使用 **`four_modules.global`**；
- 若仍无效则回退到系统内置默认 global 行。

记该行四个值为（概念上四个 M，对应四个模块键）：

```
M_基础素质, M_专业能力, M_岗位匹配, M_潜力文化   （每个在 0～1）
```

对维度 i：

```
M_i = 该行中「维度 i 所属模块 mod(i)」对应的比例
```

## 三、等级乘子 `L_i`（仅五档非零）

对 JD 原文去空格后，**按下列顺序**判断（先匹配「不重要」再匹配「重要」，避免子串误判）：

| 条件 | `L_i` |
|------|--------|
| 空 | **0** |
| 含「不重要」 | **0** |
| 含「弹性」 | **0** |
| 含「必备」，或整串为「必需」「必须」 | **0.6** |
| 含「重要」 | **0.3** |
| 含「加分项」 | **0.1** |
| **其它任意写法** | **0** |

说明：

- 仅上述五档会得到非零 L；例如仅写「加分」而无「加分项」三字 → **`L_i = 0`**。
- **`L_i = 0`** 的维度：分数仍可展示，但**不参与**综合分的分子与分母。

---

## 四、逐维权重与综合分公式

对每一个有效维度 i：

1. **合成权重**：`w_i = M_i * L_i`
2. **累加**：
   - 分子：`num = sum_i ( w_i * s_i )`（即所有维的 `w_i * s_i` 之和）
   - 分母：`den = sum_i ( w_i )`（即所有维的 `w_i` 之和）

**综合得分**（与代码一致：先除法再四舍五入到 1 位小数，再夹在 0～100）：

```
若 den > 0：
    综合得分 = round( num / den , 1 )
    综合得分 = clamp( 综合得分 , 0 , 100 )

若 den = 0：
    不产出综合分（例如全部 L_i = 0，或全部 M_i 为 0；业务上可为 null 或「不可用」）
```

其中 `round(x, 1)` 表示保留一位小数的标准四舍五入；`clamp` 表示限制在区间内。

## 五、手算示例（与审计字段对齐）

假设画像一行为：基础素质 0.2，专业能力 0.5，岗位匹配 0.2，潜力文化 0.1（和为 1）。

| 维度 | s_i | 模块 | M_i | JD 原文 | L_i | w_i | w_i * s_i |
|------|-----|------|-----|---------|-----|-----|-----------|
| A | 80 | 专业能力 | 0.5 | 必备 | 0.6 | 0.30 | 24.0 |
| B | 60 | 基础素质 | 0.2 | 不重要 | 0 | 0 | 0 |
| C | 70 | 岗位匹配 | 0.2 | 加分项 | 0.1 | 0.02 | 1.4 |

计算：

```
sum(w_i)     = 0.30 + 0 + 0.02 = 0.32
sum(w_i*s_i) = 24.0 + 0 + 1.4 = 25.4
综合得分     = round( 25.4 / 0.32 , 1 ) = round( 79.375 , 1 ) = 79.4
```

## 六、如何逐项核对（审计）

落库或返回结构中通常包含 **`portrait_meta`**（或等价字段），其中：

- **`weighting_audit`**：逐维记录 `dimension、score、category、portrait_module、M、L、w、importance_level`，可与上表对照手算；
- **`four_module_row`**（或等价）：实际使用的四大模块比例行。

用审计表中的 `M_i、L_i、w_i、s_i` 复算 `sum(w_i*s_i) / sum(w_i)` 即可验证综合得分。
