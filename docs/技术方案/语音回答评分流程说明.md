# 语音回答评分流程说明

## 功能概述

实现了完整的面试语音回答处理流程，包括语音转文字、数据库存储、智能评分和STAR追问机制。

## 流程步骤

### 1. 语音回答提交

**接口**: `POST /api/v1/interview/voice-answer`

**请求参数**:
- `session_id`: 面试会话ID（invitation_id）
- `question_id`: 问题ID
- `audio_data`: Base64编码的音频数据

### 2. 语音转文字处理

- 使用ASR服务将语音转换为文字（支持Vosk、Whisper、阿里云ASR）
- 提取音频时长等元数据
- 返回转换后的文本内容和置信度

### 3. 数据库存储

#### 3.1 双表存储策略

回答数据同时存储到两个表中以保持向后兼容：

**主存储表 - candidate_answers**:
```sql
INSERT INTO candidate_answers (
    id, session_id, question_id, answer_text, is_follow_up,
    parent_answer_id, status, create_time
) VALUES (
    'ANS_20260127170050_7BCD14BD', ?, ?, ?, false, null, 'recorded', CURRENT_TIMESTAMP
)
```

**兼容表 - interview_session**:
```sql
INSERT INTO interview_session (
    session_id, invitation_id, question_id, candidate_answer,
    session_status, question_order, audio_duration, create_time
) VALUES (
    ?, ?, ?, ?, 'IN_PROGRESS', ?, ?, CURRENT_TIMESTAMP
)
```

**字段说明**:
- `id` (candidate_answers): 答案唯一标识，格式`ANS_YYYYMMDDHHMMSS_RANDOM`
- `session_id`: 会话唯一标识（关联interview_session.session_id）
- `question_id`: 问题ID（关联interview_question.question_id）
- `answer_text`: ASR转换后的回答文本
- `is_follow_up`: 是否为追问答案
- `parent_answer_id`: 父答案ID（追问时指向主问题答案）
- `status`: 状态（recorded/evaluated/pending）
- `audio_duration`: 音频时长（秒）

### 4. 智能评分流程

#### 4.1 获取评估要点

根据`question_id`从`interview_question`表获取`evaluation_points`（JSONB格式）：

```sql
SELECT iq.evaluation_points, iq.question_type
FROM interview_question iq
WHERE iq.question_id = ?
```

#### 4.2 LLM评分算法

采用**差异化模板评分法**，根据问题类型选择不同的评分模板：

##### 支持的问题类型
- `BASIC_INFO`: 基础信息类问题（自我介绍、职业规划等）
- `PROFESSIONAL`: 专业技能类问题（技术深度、项目经验等）

##### 评分流程

1. **动态解析评估要点**
   - 支持多种格式：JSON字符串、JSON对象、列表
   - 根据问题类型自动标准化要点格式

2. **差异化提示词模板**
   - 基础信息题：注重表达清晰度和基本信息完整性
   - 专业技能题：注重技术深度和解决思路的合理性

3. **LLM评分执行**
   - 使用独立的`EvaluationService`服务
   - 设置较低温度(0.1)确保评分稳定性
   - 最大token限制1500以控制响应长度

4. **结果解析与验证**
   - JSON格式解析和字段验证
   - 自动分数范围校正(0-100)
   - 智能等级判断和标准化

##### 评分维度和权重
```json
{
  "score": 85,
  "grade": "优",
  "point_scores": [
    {
      "point": "技术方案的合理性分析",
      "score": 1.0,
      "weight": 0.4
    },
    {
      "point": "实现细节的完整性",
      "score": 0.8,
      "weight": 0.3
    }
  ],
  "reasoning": "候选人展现了扎实的技术基础和良好的问题分析能力",
  "need_follow_up": false,
  "follow_up_question": null
}
```

#### 4.3 数据库存储评估结果

评分结果存储到`candidate_answers`表：

```sql
UPDATE candidate_answers SET
    evaluation_result = ?::jsonb,
    point_evaluations = ?::jsonb,
    final_score = ?,
    need_follow_up = ?,
    follow_up_question = ?,
    follow_up_evaluation_points = ?::jsonb,
    status = 'evaluated',
    update_time = CURRENT_TIMESTAMP
WHERE id = ?
```

### 5. STAR追问机制

#### 5.1 STAR维度分析
基于Situation(情境)、Task(任务)、Action(行动)、Result(结果)的追问机制：

- **Situation**: 问题发生的具体情境
- **Task**: 明确的职责和目标
- **Action**: 具体采取的行动步骤
- **Result**: 取得的成果和影响

#### 5.2 智能追问触发
```json
{
  "action": "follow_up",
  "followup_question": "请详细描述您在项目中具体采用了哪些技术方案？",
  "missing_dimension": "ACTION",
  "follow_up_evaluation_points": [
    {
      "point": "详细说明具体行动步骤",
      "weight": 0.4
    },
    {
      "point": "描述使用的方法和技术",
      "weight": 0.3
    }
  ]
}
```

#### 5.3 追问答案处理
- 追问答案存储为独立记录，关联到主答案
- 对追问答案单独进行评分
- 计算综合得分（主问题+追问的加权平均）

## 技术实现要点

### 1. 数据库操作
- **双表存储**: 同时写入`candidate_answers`和`interview_session`表
- **答案链管理**: 通过`parent_answer_id`建立追问关系
- **状态机管理**: `recorded` → `evaluated` → `pending`状态流转
- **向后兼容**: 保持现有API接口不变

### 2. LLM服务集成
- **专用评分服务**: 使用独立的`EvaluationService`类
- **差异化模板**: 根据问题类型加载不同的提示词模板
- **智能解析**: 支持多种评估要点格式的动态解析
- **结果验证**: 自动验证和标准化评分结果

### 3. STAR追问机制
- **维度分析**: 基于STAR模型分析回答完整性
- **智能触发**: 根据评分结果和维度缺失情况触发追问
- **要点生成**: 针对缺失维度生成专门的评估要点
- **答案关联**: 追问答案与主答案建立关联关系

### 4. 错误处理
- **ASR失败处理**: 置信度检查和失败回退
- **评分异常处理**: 默认评分结果和详细错误日志
- **数据库异常处理**: 事务回滚和状态一致性保证
- **网络异常处理**: 重试机制和超时控制

### 5. 性能优化
- **异步处理**: ASR和LLM调用全部异步化
- **连接池**: 数据库连接池复用
- **缓存机制**: 提示词模板缓存
- **并发控制**: 支持多用户同时面试

## 注意事项

1. **数据库约束**: 采用双表存储策略，`candidate_answers`为主表，`interview_session`为兼容表
2. **评分准确性**: LLM评分结果受到模型质量、提示词设计和评估要点质量影响
3. **并发处理**: 支持多用户同时进行面试，需注意数据库锁竞争
4. **数据一致性**: 通过事务保证双表数据一致性，状态机确保流程正确性
5. **追问机制**: STAR追问会延长面试时长，但能显著提升评估质量

## 扩展建议

1. **评估结果持久化**: ✅ 已实现 - 评分结果存储在`candidate_answers`表的`evaluation_result`字段
2. **评分模型优化**: 可根据行业特点定制评分模板和权重算法
3. **多轮对话**: ✅ 已实现 - STAR追问机制支持多轮深度对话
4. **批量评估**: 可扩展支持历史回答的批量重新评估功能
5. **实时反馈**: 可增加面试过程中的实时评分反馈功能
6. **评估报告**: 可生成详细的面试评估报告和改进建议

## 数据流图

```
语音数据 → ASR转换 → 文本存储(candidate_answers)
                      ↓
                LLM智能评分 → 结果存储
                      ↓
                STAR维度分析 → 触发追问/进入下一题
                      ↓
                追问答案处理 → 更新主答案评估
                      ↓
                综合评分计算 → 面试完成
```
