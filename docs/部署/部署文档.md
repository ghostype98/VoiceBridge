# VoiceBridge 部署文档

本文档说明如何在服务器或本机从零部署 VoiceBridge 语音交互服务，每一步均配有简要说明；**包含生产环境开机自启（systemd）**配置。

---

## 一、环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Linux（推荐 Ubuntu 20.04+）或 macOS |
| Python | 3.8 及以上 |
| Conda | Anaconda3 或 Miniconda3（用于管理 Python 环境） |
| 端口 | 默认 8002（主服务）、8003（WebSocket），以 `config/config.yaml` 为准，确保未被占用 |

**说明**：项目通过 Conda 使用名为 `datastore` 的环境运行，请先安装 Conda。

---

## 二、第一步：获取项目代码

将项目克隆或拷贝到目标目录，例如：

```bash
# 使用 Git 克隆（仓库地址）
git clone https://github.com/YOUR_ORG/voicebridge.git /opt/voicebridge
cd /opt/voicebridge
```

**说明**：仓库地址为 `https://github.com/YOUR_ORG/voicebridge.git`；可替换为实际部署目录。后续所有脚本路径均相对于项目根目录（即包含 `app`、`bash`、`config` 的目录）。

---

## 三、第二步：创建并激活 Conda 环境

项目使用 **datastore** 环境。若尚未创建，可先创建再安装依赖；若已存在则直接激活。

```bash
# 创建环境（仅首次需要）
conda create -n datastore python=3.10 -y

# 激活环境（每次新开终端都需要执行）
conda activate datastore
```

**说明**：`conda activate datastore` 后，终端提示符前会显示 `(datastore)`，表示当前已在该环境中。

---

## 四、第三步：安装 Python 依赖

在项目根目录下、且已激活 `datastore` 环境时执行：

```bash
cd /opt/voicebridge
pip install -r requirements.txt
```

**说明**：  
- `requirements.txt` 包含 FastAPI、uvicorn、阿里云 SDK、WebSocket、日志等依赖。  
- 若提示权限错误，可加 `--user` 或使用虚拟环境。  
- 安装完成后可用 `python -c "import fastapi, uvicorn; print('ok')"` 快速验证。

---

## 五、第四步：配置文件（端口唯一修改处）

**端口与核心服务配置均只需修改一个文件：`config/config.yaml`**。修改该文件中的端口后，重启服务即可；启动脚本与 NATAPP 会自动从该文件读取主端口。

主配置文件为 **`config/config.yaml`**。部署时需要至少确认以下部分。

### 5.1 语音流式服务（ASR）

- **路径**：`config/config.yaml` → `voice_streaming.asr`
- **必填**：阿里云 ASR 的 `appkey`、`access_key_id`、`access_key_secret`
- **说明**：用于实时语音转文字。可从阿里云控制台创建/获取；敏感信息也可通过环境变量或 `config/credentials.yaml` 配置（见 `auto_config` 段）。

### 5.2 主服务端口（唯一端口配置处）

- **路径**：`config/config.yaml` → `services.voicebridge.port` 与 `server.port`（二者需一致）
- **默认**：`host: "0.0.0.0"`、`port: 8002`
- **说明**：主服务端口同时提供 HTTP API 与前端静态页面，无需单独起前端服务。WebSocket 端口见 `voice_streaming.websocket.port`（默认 8003）。**以后更换端口只需改本文件上述三处。**

### 5.3 对话与 LLM（可选）

- **Rasa**：`rasa.endpoint`、`rasa.port`（默认 8012），若不需要对话管理可设 `enabled: false`。
- **LLM**：`llm.api_base`、`llm.model` 等，按实际使用的模型服务填写。

### 5.4 日志与存储

- 日志会写入 `logs/` 下各子目录（如 `logs/service/`、`logs/app/`），无需在 config 里改即可使用。
- 存储目录见 `storage.path`、`storage.audio_path`，默认为 `./storage`、`./storage/audio`。

**说明**：修改配置后需重启服务生效。

---

## 六、第五步：前端静态资源（可选）

若项目根目录下存在 **`frontend`** 目录，主服务会自动在主服务端口（默认 8002）提供登录、面试等页面（`/`、`/login`、`/interview` 等）。

- 若没有 `frontend` 目录，仅 API 可用，前端页面不可用。  
- 若有，无需再单独启动前端开发服务器，直接访问 `http://<主机>:8002` 即可（端口以 config 为准）。

---

## 七、第六步：启动服务

在项目根目录执行：

```bash
bash /opt/voicebridge/bash/start_service.sh
```

**脚本会依次执行**：  
1. 解析项目根目录与日志目录。  
2. 自动查找并 source Conda（如 `~/anaconda3/etc/profile.d/conda.sh` 或 miniconda 路径）。  
3. 执行 `conda activate datastore`。  
4. 设置 `PYTHONPATH` 为项目根目录。  
5. 检查 `voicebridge.pid`：若已有对应进程在跑则提示“服务已在运行”并退出。  
6. 使用 `nohup python services/run.py` 后台启动服务，并将 PID 写入 `voicebridge.pid`。  
7. 等待约 3 秒后检查进程是否存在，并输出访问地址与日志路径。

**成功时终端会看到类似**（端口以 config/config.yaml 为准，默认 8002/8003）：  
- 主服务: `http://0.0.0.0:8002`  
- WebSocket: `ws://0.0.0.0:8003`  
- 前端: `http://localhost:8002/login`  
- API 文档: `http://localhost:8002/docs`  
- 日志: `logs/service/service_<时间戳>.log`

---

## 八、第七步：验证部署

1. **健康检查**  
   ```bash
   curl http://localhost:8002/health
   ```  
   应返回 `{"status":"healthy","service":"VoiceBridge"}`。

2. **查看日志**  
   ```bash
   tail -f /opt/voicebridge/logs/service/service_*.log
   ```  
   最新日志文件名为 `service_<时间戳>.log`，可在 `logs/service/` 下用 `ls -t` 查看最新一个。

3. **访问 API 文档**  
   浏览器打开 `http://<本机IP或localhost>:8002/docs`，可查看并调试接口（端口以 config 为准）。

4. **访问前端**  
   若有 `frontend`，访问 `http://<本机IP或localhost>:8002/login` 进行登录与面试流程。

---

## 九、服务管理命令

| 操作 | 命令 |
|------|------|
| 启动 | `bash /opt/voicebridge/bash/start_service.sh` |
| 停止 | `bash /opt/voicebridge/bash/stop_service.sh` |
| 重启 | `bash /opt/voicebridge/bash/restart_service.sh` |
| 状态 | `bash /opt/voicebridge/bash/status_service.sh` |

**说明**：  
- 停止脚本会读取 `voicebridge.pid`，先 `kill`，若未退出则 `kill -9`，并删除 PID 文件。  
- 若 PID 文件丢失，停止脚本会尝试通过 `python.*services/run.py` 查找进程并提示是否结束。

## 十、内网穿透（可选）

若需从外网访问部署在内网的服务，可使用 NATAPP。

1. **准备**  
   - 在 [natapp.cn](https://natapp.cn) 注册并创建隧道，获得 `authtoken`。  
   - 将 `tools/natapp/config.ini` 中的 `authtoken` 改为你的 token。  
   - `tools/natapp/start.sh` 会从 `config/config.yaml` 读取主服务端口并写入 `config.ini` 的 `lanport`，无需手动改 config.ini。

2. **启动 NATAPP**  
   ```bash
   bash /opt/voicebridge/tools/natapp/start.sh
   ```  
   日志写入 `logs/natapp/natapp.log`，可用 `tail -f logs/natapp/natapp.log` 查看。

3. **停止**  
   ```bash
   bash /opt/voicebridge/tools/natapp/stop.sh
   ```

**说明**：先确保本机主服务（默认 8002）已启动，再启动 NATAPP；外网访问使用 NATAPP 提供的域名（如示例中的 `http://recruitment.natapp1.cc`）。

---

## 十一、常见问题

- **“无法找到 conda”**  
  安装 Anaconda/Miniconda 并将 `conda.sh` 放到脚本可找到的路径（如 `~/anaconda3/etc/profile.d/conda.sh`），或保证 `conda` 在 `PATH` 中。

- **“无法激活 datastore 环境”**  
  先执行 `conda create -n datastore python=3.10 -y`，再执行启动脚本。

- **“服务已在运行”**  
  已有进程在跑，如需重启请先执行 `bash bash/stop_service.sh`，再执行 `start_service.sh`。

- **端口被占用**  
  只需修改 `config/config.yaml` 中 `services.voicebridge.port`、`server.port` 及 `voice_streaming.websocket.port` 为未占用端口；NATAPP 启动时会自动从 config 读取主端口。

- **ASR 不可用**  
  检查 `config/config.yaml` 中 `voice_streaming.asr` 的 appkey、access_key_id、access_key_secret 是否正确，以及网络是否可达阿里云。

## 十二、生产环境与开机自启动（systemd）

生产环境使用 systemd 托管**主服务**与**内网穿透**，实现开机自启、异常重启与统一日志。  
当前 unit 示例：**项目路径** `/opt/voicebridge`，**运行用户** `voicebridge`。部署到其他机器时再修改各 `.service` 中的路径与 `User`/`Group`。

### 12.1 一键安装（主服务 + 内网穿透均开机自启）

在项目根目录执行：

```bash
cd /opt/voicebridge

# 1. 安装两个服务单元
sudo cp deploy/voicebridge.service /etc/systemd/system/
sudo cp deploy/natapp.service /etc/systemd/system/

# 2. 重载并启用开机自启
sudo systemctl daemon-reload
sudo systemctl enable voicebridge
sudo systemctl enable natapp

# 3. 立即启动（先主服务，再内网穿透）
sudo systemctl start voicebridge
sudo systemctl start natapp

# 4. 确认状态
sudo systemctl status voicebridge
sudo systemctl status natapp
```

### 12.2 常用命令

| 操作 | 命令 |
|------|------|
| 主服务状态 / 重启 / 停止 | `sudo systemctl status voicebridge` / `restart` / `stop` |
| 主服务日志 | `journalctl -u voicebridge -f` |
| 内网穿透状态 / 启动 / 停止 | `sudo systemctl status natapp` / `start` / `stop` |
| 内网穿透日志 | `journalctl -u natapp -f` 或 `tail -f logs/natapp/natapp.log` |
| 关闭开机自启 | `sudo systemctl disable voicebridge` / `sudo systemctl disable natapp` |

**说明**：systemd 通过 `bash/voicebridge-systemd.sh` 前台启动主服务，无需 PID 文件；与手动执行 `bash bash/start_service.sh` 二选一，不要同时用两种方式启动。内网穿透（NATAPP）会在主服务之后自动启动。

---

## 十三、生产上线检查项

上线前建议逐项确认：

| 检查项 | 说明 |
|--------|------|
| **端口** | `config/config.yaml` 中主服务端口（默认 8002）、WebSocket（默认 8003）与防火墙/安全组一致；若对外暴露，需放行对应端口。 |
| **配置** | `server.debug` 为 `false`；ASR、LLM、Rasa 等地址与密钥正确；敏感信息建议用环境变量或 `credentials.yaml`，勿提交到仓库。 |
| **Conda** | 运行用户下已安装 Conda，且存在 `datastore` 环境、依赖已 `pip install -r requirements.txt`。 |
| **开机自启** | 已按第十二节配置 systemd，并 `systemctl enable voicebridge`、`systemctl enable natapp`（主服务 + 内网穿透均开机自启）。 |
| **日志** | 日志目录 `logs/` 可写；重要环境可配合 logrotate 做日志轮转。 |
| **健康检查** | 部署后访问 `http://<本机>:8002/health` 返回 `{"status":"healthy"}`。 |
| **NATAPP** | 官网隧道「本地端口」与 config 主服务端口一致；authtoken 有效；若外网访问则必配。 |

---

以上为完整部署步骤与说明，按顺序执行即可在本机或服务器上运行 VoiceBridge 服务；生产环境请完成第十二、十三节。
